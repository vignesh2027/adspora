## 1. What does this tool do?

Adspora is a creative-fatigue command center for a performance/affiliate media-buying team. It closes the loop a media buyer runs manually every single day — **Detect → Diagnose → Generate** — in seconds:

- **Detect.** It pulls performance data from every platform (Meta, Google, Taboola, TikTok), normalizes the differently-named metrics into one schema, and scores every creative **0–100** on a deterministic, explainable fatigue model built from ROAS decay vs. the creative's own peak, CTR trend, distance from breakeven ROAS, and creative age.
- **Diagnose.** For any fatiguing creative, an AI analyst explains the **root cause** in a media buyer's language — audience saturation, hook wear-out, trend decay, auction pressure — references the actual numbers, sets an urgency level, and recommends the next angle to test.
- **Generate.** It writes **3 launch-ready replacement ads** per creative, each tuned to that platform's native voice (TikTok creator-style, Google search-intent, Taboola advertorial, Meta scroll-stopping hook), with a one-click copy button so the buyer ships the next test immediately.

On top of that loop: a real **Overview** dashboard (KPIs, blended ROAS over time, fatigue-by-creative, and a "top creatives to replace now" spend-at-risk panel), a sortable/filterable **Creatives** table, a **Reports** view with CSV export, and a **CSV adapter** designed so the same engine plugs straight into real ad-platform APIs.

## 2. Why did you build THIS one?

Because in 2026 the performance bottleneck is no longer targeting — **it's creative.** Creatives fatigue in 7–14 days, CPMs keep rising, and the team's edge comes down to two things: *spotting fatigue before it bleeds budget*, and *shipping the next winner faster than the competition.* Both are currently manual, repetitive, and exactly the kind of judgment work AI is now good enough to own.

For a lean, high-spend affiliate team that advertises at scale across Google, Meta, Taboola and TikTok, this isn't a nice-to-have — it's the difference between protecting margin and watching it erode in real time. I picked the single most valuable, most universal problem in the workflow and built the tool that turns a reactive grind into a proactive system.

## 3. What would you build next if this were my full-time job?

1. **Live API connectors** — replace the seeded/CSV layer with real OAuth connectors to the Meta, Google Ads, TikTok and Taboola reporting APIs (the adapter layer is already designed for this drop-in).
2. **Predictive fatigue** — move from "this creative *is* fatiguing" to "this creative *will* cross breakeven in ~3 days," using each creative's decay curve to forecast and pre-stage replacements.
3. **Closed-loop launch** — push generated variations straight back to the ad platforms as paused drafts, and auto-promote the winner once it clears a ROAS threshold.
4. **Creative DNA analysis** — tag what *kind* of hook/angle/format works per platform and audience, so generation gets smarter with every campaign instead of starting cold.

## Architecture & how to run

This live build runs **fully client-side** (static, deployable to GitHub Pages with zero backend or API keys) so the demo is instant and nothing can break:

- **`docs/index.html`** — the entire app (Emerald Noir UI, landing + dashboard + AI Studio + reports), built with vanilla JS and Chart.js. Clean separation between the **fatigue engine**, the **AI diagnosis/generation engine**, and the **render layer**.
- **`docs/seed.json`** — 24 realistic creatives across 4 platforms, each with ~30 days of daily metrics that ramp, peak and decay — generated by the exact fatigue-scoring logic.

The repo also contains the **full-stack version** (`backend/` FastAPI + MongoDB, `frontend/` React) with live LLM-powered diagnosis/generation, for the production architecture the static demo mirrors.

```bash
# Run the live demo locally
cd docs && python3 -m http.server 8000
# open http://localhost:8000
```
